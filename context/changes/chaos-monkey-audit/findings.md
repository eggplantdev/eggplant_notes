### Navigation & cross-cutting

| #       | Sev          | Finding                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| ------- | ------------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **X-1** | 🟠 **Major** | **React hydration error #418 on `/notes`** (reproduces on every load). Root cause: `formatLocaleDate()` in `src/lib/utils/date.ts:78` is `new Date(value).toLocaleDateString()` with **no fixed locale or `timeZone`**. SSR renders with the server's locale + UTC (`6/10/2026`), client re-renders with the browser's locale + zone (`11.06.2026`) → text-content mismatch → #418. Ironically the same file pins `timeZone`/`en-CA` for its heatmap/streak helpers (with comments explaining exactly this hazard); the two `formatLocale*` helpers were left zone/locale-dependent. Fix: pin a locale + `APP_TIME_ZONE` (already defined as `Europe/Warsaw` in the file) in `formatLocaleDate`/`formatLocaleDateTime`, or render those dates client-only. Console stack: `Minified React error #418 … args[]=text`. |
| X-2     | 🟡           | **Inconsistent date format across views** (related to X-1): notes **list** shows `DD.MM.YYYY` (`11.06.2026`), note **detail** shows `M/D/YYYY` (`6/10/2026`) — different formatters, and an apparent **timezone off-by-one** (list shows the 11th for notes created on the 10th). Pinning one zone+format fixes both X-1 and X-2.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| X-3     | 🟢           | **OpenRouter "Connect" button works.** Clicking the nav button (logged-in, no key) redirects to the real OpenRouter **PKCE OAuth** flow: `openrouter.ai/auth?callback_url=http://localhost:3250/api/openrouter/callback&code_challenge=…&code_challenge_method=S256`. Correct authorize-code+S256 wiring; callback points back to `/api/openrouter/callback`. (Did not complete OAuth — owner tests the AI path.) ## Top fixes, in priority order

1. **X-1 — `/notes` hydration error (#418).** Pin locale + `APP_TIME_ZONE` in `formatLocaleDate`/`formatLocaleDateTime` (`src/lib/utils/date.ts:78-84`). One-line-ish fix; also resolves X-2. Highest value: it's a real React error on a core page.
2. **R-1 — review answer reveal doesn't reset.** Reset the `showAnswer` state when the current card changes in the review widget. Defeats spaced-repetition recall — core-feature correctness.
3. **N-1 — inline code renders literal backticks.** Fix the shared markdown renderer's inline-code handling (affects notes + cards everywhere).
4. **U-4 — duplicated validation messages.** Dedupe field errors or drop the redundant `onSubmit` validator across the TanStack forms.
5. **M-7 — note-form label wrap on mobile.** Cosmetic; stack the "Memory cards" label above its buttons below ~`sm`.
